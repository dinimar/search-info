				Урок 18. Меняем layoutParams в рабочем приложении</a>
			</h1>

			</header>
    
    
  <!-- Aside -->

	<section class="article-content clearfix">
		<p>В этом уроке мы:</p>
<p>- изменяем layout-параметры для уже существующих компонентов экрана</p>
 
<p>Мы умеем создавать экранные компоненты и настраивать для них расположение с помощью <strong>LayoutParams</strong>. В этом уроке разберемся, как изменять layout-параметры уже существующих компонентов.</p>
<p>Менять мы будем вес – <strong>weight</strong>. Нарисуем <strong>SeekBar </strong>(регулятор или «ползунок») и <strong>две кнопки</strong>. И будем <strong>регулировать </strong>пространство занимаемое кнопками, используя параметр веса.</p>
<p> </p>
<p>Создадим проект:</p>
<p><strong>Project name</strong>: P0181_DynamicLayout3<br/> <strong>Build Target</strong>: Android 2.3.3<br/> <strong>Application name</strong>: DynamicLayout3<br/> <strong>Package name</strong>: ru.startandroid.develop.dynamiclayout3<br/> <strong>Create Activity</strong>: MainActivity</p>
<p> </p>
<p>Открываем <strong>main.xml</strong> и создаем такой экран:</p>
<pre class="brush:xml">&lt;?xml version="1.0" encoding="utf-8"?&gt;
&lt;LinearLayout
    xmlns:android="http://schemas.android.com/apk/res/android"
    android:orientation="vertical"
    android:layout_width="fill_parent"
    android:layout_height="fill_parent"&gt;
    &lt;SeekBar
        android:layout_height="wrap_content"
        android:layout_width="match_parent"
        android:max="100"
        android:progress="50"
        android:layout_marginTop="20dp"
        android:id="@+id/sbWeight"&gt;
    &lt;/SeekBar&gt;
    &lt;LinearLayout
        android:id="@+id/linearLayout1"
        android:layout_width="match_parent"
        android:orientation="horizontal"
        android:layout_height="wrap_content"
        android:layout_marginTop="30dp"&gt;
        &lt;Button
            android:layout_height="wrap_content"
            android:id="@+id/btn1"
            android:text="Button1"
            android:layout_weight="1"
            android:layout_width="wrap_content"&gt;
        &lt;/Button&gt;
        &lt;Button
            android:layout_height="wrap_content"
            android:id="@+id/btn2"
            android:text="Button2"
            android:layout_weight="1"
            android:layout_width="wrap_content"&gt;
        &lt;/Button&gt;
    &lt;/LinearLayout&gt;
&lt;/LinearLayout&gt;</pre>
<p> </p>
<p>Мы используем компонент SeekBar. Он похож на полосу прокрутки и позволяет задавать какое-либо значение из диапазона. У этого компонента есть свойства <strong>max </strong>и <strong>progress</strong>. <strong>Max </strong>– это какое значение выдает SeekBar, когда он выкручен на <strong>максимум</strong>. <strong>Progress </strong>– это <strong>текущее значение </strong>ползунка. Максимум сделаем = <strong>100</strong>, а текущее значение будет на половине – <strong>50</strong>.</p>
<p>Кнопки у нас с шириной по содержимому и <strong>вес для обоих = 1</strong>. Они <strong>поровну </strong>делят пространство LinearLayout, в котором находятся.</p>
<p>Осталось только написать нужный код, чтобы все заработало. Открываем <strong>MainActivity.java</strong>, опишем и найдем компоненты и получим доступ к их LayoutParams.</p>
<div class="java">
<pre class="brush:java">public class MainActivity extends Activity {

  SeekBar sbWeight;
  Button btn1;
  Button btn2;

  LinearLayout.LayoutParams lParams1;
  LinearLayout.LayoutParams lParams2;

  public void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.main);

    sbWeight = (SeekBar) findViewById(R.id.sbWeight);

    btn1 = (Button) findViewById(R.id.btn1);
    btn2 = (Button) findViewById(R.id.btn2);

    lParams1 = (LinearLayout.LayoutParams) btn1.getLayoutParams();
    lParams2 = (LinearLayout.LayoutParams) btn2.getLayoutParams();
  }
}</pre>
<p> </p>
</div>
<p>Мы используем метод <a href="http://developer.android.com/reference/android/view/View.html#getLayoutParams()" target="_blank">getLayoutParams</a> для получения LayoutParams компонента. Но этот метод возвращает базовый <strong>ViewGroup.LayoutParams</strong>, а нам нужен<strong> LinearLayout.LayoutParams</strong>, поэтому делаем преобразование. В результате - <strong>lParams1 </strong>и <strong>lParams2 </strong>теперь являются <strong>LayoutParams </strong>для компонентов <strong>btn1 </strong>и <strong>btn2</strong>. Т.е. работая, например, с lParams1 мы влияем на btn1. Сейчас мы это используем.</p>
<p>Для SeekBar нужен будет обработчик, который будет реагировать на изменения. Это мы поручим Activity. Для этого надо добавить к описанию класса<strong> implements OnSeekBarChangeListener</strong>:</p>
<div class="java">
<pre class="brush:java">public class MainActivity extends Activity implements OnSeekBarChangeListener {</pre>
<p> </p>
</div>
<div class="java">А также надо добавить методы обработчика, которые теперь обязана реализовывать Activity.</div>
<div class="java">
<pre class="brush:java">  @Override
  public void onProgressChanged(SeekBar seekBar, int progress,
      boolean fromUser) {

  }

  @Override
  public void onStartTrackingTouch(SeekBar seekBar) {

  }

  @Override
  public void onStopTrackingTouch(SeekBar seekBar) {

  }</pre>
<p> </p>
<p>Обработчик содержит три метода. Из названий понятно, что: </p>
</div>
<p>- <a href="http://developer.android.com/reference/android/widget/SeekBar.OnSeekBarChangeListener.html#onStartTrackingTouch(android.widget.SeekBar)" target="_blank">onStartTrackingTouch</a> срабатывает, когда начинаем тащить ползунок<br/>- <a href="http://developer.android.com/reference/android/widget/SeekBar.OnSeekBarChangeListener.html#onProgressChanged(android.widget.SeekBar, int, boolean)" target="_blank">onProgressChanged</a> срабатывает все время, пока значение меняется<br/>- <a href="http://developer.android.com/reference/android/widget/SeekBar.OnSeekBarChangeListener.html#onStopTrackingTouch(android.widget.SeekBar)" target="_blank">onStopTrackingTouch</a> срабатывает, когда отпускаем ползунок</p>
<p> </p>
<p>Мы будем использовать метод onProgressChanged. Так изменения будут видны во время перетаскивания ползунка.</p>
<div class="java">
<pre class="brush:java">  @Override
  public void onProgressChanged(SeekBar seekBar, int progress,
      boolean fromUser) {
    int leftValue = progress;
    int rightValue = seekBar.getMax() - progress;
    // настраиваем вес
    lParams1.weight = leftValue;
    lParams2.weight = rightValue;
    // в текст кнопок пишем значения переменных
    btn1.setText(String.valueOf(leftValue));
    btn2.setText(String.valueOf(rightValue));
  }</pre>
<p> </p>
</div>
<div class="java">переменная <strong>leftValue </strong>– текущее значение SeekBar, т.е. то что слева от ползунка</div>
<p>переменная <strong>rightValue </strong>– то, что справа от ползунка, т.е. из максимума вычесть текущее значение.</p>
<p>Соответственно эти значения и используем как вес. Чем ползунок <strong>левее</strong>, тем <strong>меньше leftValue</strong> и <strong>больше rightValue</strong>, а значит <strong>меньше ширина btn1</strong> и <strong>больше ширина btn2</strong>. И наоборот.</p>
<p><img src="https://lh4.googleusercontent.com/-bj7qZmZ76Ok/TneFSHVUE-I/AAAAAAAAAVA/xNX1YXQfLL4/s800/20110919_L0018_L_layoutFile.jpg" alt="" width="280" height="456" border="0"/></p>
<p>Также для наглядности в текст кнопок будем записывать значения переменных.</p>
<p>Ну и конечно не забываем, что надо обработчик (Activity) присвоить View-компоненту, события которого необходимо обрабатывать:</p>
<div class="java">
<pre class="brush:java">    setContentView(R.layout.main);

    sbWeight = (SeekBar) findViewById(R.id.sbWeight);
    sbWeight.setOnSeekBarChangeListener(this);

    btn1 = (Button) findViewById(R.id.btn1);</pre>
<p> </p>
</div>
<div class="java">(Обратите внимание. Я ввожу новый прием подачи кода. То, что подчеркнуто – это новый код, а обычный шрифт – уже существующий код. Вам надо найти существующий код и дописать к нему новый, чтобы получился этот фрагмент.)</div>
<p> </p>
<p>Все сохраним и запустим приложение. Перетаскивая ползунок, меняем размеры кнопок:</p>
<p><img src="https://lh4.googleusercontent.com/-ByxVcSLLK2U/TneFSUWLjKI/AAAAAAAAAVE/P2CS9R29JMk/s640/20110919_L0018_L_screen1.jpg" alt="" width="640" height="454" border="0"/></p>
<p> </p>
<p>Выглядит эффектно, я считаю ) И кода - всего несколько строк.</p>
<p>Есть небольшой нюанс. Как верно заметили в каментах, если просто написать код lParams1.weight = 1, то компонент не изменится. Необходимо дописать код: btn1.requestLayout(). Тогда кнопка прочтет Layout и перерисуется. Этот метод уже вызывается в setText, поэтому мы его здесь явно не вызываем.</p>
<p> </p>
<p>Теперь мы знаем достаточно много, и на следующих уроках попробуем написать первое осмысленное приложение – калькулятор.</p>
<p> </p>
<p>Полный код урока:</p>
<div class="java">
<pre class="brush:java">public class MainActivity extends Activity implements OnSeekBarChangeListener {

  SeekBar sbWeight;
  Button btn1;
  Button btn2;

  LinearLayout.LayoutParams lParams1;
  LinearLayout.LayoutParams lParams2;

  /** Called when the activity is first created. */
  @Override
  public void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.main);

    sbWeight = (SeekBar) findViewById(R.id.sbWeight);
    sbWeight.setOnSeekBarChangeListener(this);

    btn1 = (Button) findViewById(R.id.btn1);
    btn2 = (Button) findViewById(R.id.btn2);

    lParams1 = (LinearLayout.LayoutParams) btn1.getLayoutParams();
    lParams2 = (LinearLayout.LayoutParams) btn2.getLayoutParams();
  }

  @Override
  public void onProgressChanged(SeekBar seekBar, int progress,
      boolean fromUser) {
    int leftValue = progress;
    int rightValue = seekBar.getMax() - progress;
    // настраиваем вес
    lParams1.weight = leftValue;
    lParams2.weight = rightValue;
    // в текст кнопок пишем значения переменных
    btn1.setText(String.valueOf(leftValue));
    btn2.setText(String.valueOf(rightValue));
  }

  @Override
  public void onStartTrackingTouch(SeekBar seekBar) {
  }

  @Override
  public void onStopTrackingTouch(SeekBar seekBar) {
  }
}</pre>
<p> </p>
</div>

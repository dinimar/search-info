	<section class="article-content clearfix">
		<p>This lesson:</p>
<p>- explore Uri and data intent attribute;<br/>- launch system apps (browser, dialer, map).</p>
 
<p> </p>
<p>We know that <strong>Intent</strong> has an <strong>action</strong> attribute. It is used to indicate action: <strong>viewing</strong> or <strong>editing</strong> for example. It’s common, that action is often applied to smth,so we need to declare an <strong>object</strong>, but not only an action, and Intent has <strong>data</strong> attribute for it.</p>
<p>A way to set value to this attribute is Intent’s <a href="http://developer.android.com/reference/android/content/Intent.html#setData(android.net.Uri)" target="_blank" rel="noopener noreferrer">setData (Uri data)</a> method, and <a href="http://developer.android.com/reference/android/net/Uri.html" target="_blank" rel="noopener noreferrer">Uri</a> object must be passed to this method.</p>
<p>Uri is an object which uses string, disassembles it to parts and remember it. We are not allowed to use any string we want, we can only use strings which are declared according to this document: <a href="http://www.faqs.org/rfcs/rfc2396.html" target="_blank" rel="noopener noreferrer">RFC 2396</a>. Uri has lot’s of methods which allow us to retrieve separate parts from parsed string.</p>
<p>I will create Uri from string and pass (using colon) method’s name and a value which it returns to logs. Let’s take this string, http address for example:</p>
<div class="java">
<pre class="brush:java">Uri uri = Uri.parse("http://developer.android.com/reference/android/net/Uri.html");</pre>
</div>
<p>Let’s look at what methods return:</p>
<p><em>uri.getScheme(): http<br/></em><em>uri.getSchemeSpecificPart(): //developer.android.com/reference/android/net/Uri.html<br/></em><em>uri.getAuthority(): developer.android.com<br/></em><em>uri.getHost(): developer.android.com<br/></em><em>uri.getPath(): /reference/android/net/Uri.html<br/></em><em>uri.getLastPathSegment(): Uri.html</em></p>
<p>Scheme, Authority, Host, Path, and others are concepts, taken from RFC document I linked before. Following the link you can find full description of them, understand their meaning and verify with what was returned by Uri.</p>
<p> </p>
<p>Let’s take a look at some more example:</p>
<p> </p>
<p>FTP</p>
<pre class="brush:java">Uri uri = Uri.parse("ftp://bob @ google.com:80/data/files");</pre>
<p>(<em>As for real, the code above is written in one string. There are spaces before and after @ because of layout features</em>)</p>

<p> </p>
<p>Coordinates</p>
<div class="java">
<pre class="brush:java">Uri uri = Uri.parse("geo:55.754283,37.62002"); </pre>
</div>
<p><em>uri.getScheme(): geo<br/>uri.getSchemeSpecificPart(): 55.754283,37.62002</em></p>
<p>We only could retrieve Scheme and SchemeSpecificPart here.</p>
<p> </p>
<p>Phone number</p>
<div class="java">
<pre class="brush:java">Uri uri = Uri.parse("tel:12345");</pre>
</div>
<p><em>uri.getScheme(): tel<br/>uri.getSchemeSpecificPart():12345</em></p>
<p>Similarly, only two parts we could retrieve here.</p>
<p> </p>
<p>Contact</p>
<div class="java">
<pre class="brush:java">Uri uri = Uri.parse("content://contacts/people/1");</pre>
</div>
<p><em>uri.getScheme(): content<br/>uri.getSchemeSpecificPart(): //contacts/people/1<br/>uri.getAuthority(): contacts<br/>uri.getPath(): /people/1<br/>uri.getLastPathSegment(): 1</em></p>
<p>Scheme equals to content within this example. It’s a special data type - Content Provider which allows any program to give access to it’s data, and other programs to read and edit this data. We will unveil this topic later and create this data type by ourselves.</p>
<p>You can inquire standard supported Uris <a href="http://developer.android.com/guide/appendix/g-app-intents.html" target="_blank" rel="noopener noreferrer">here</a></p>
<p> </p>
<p>Examples show that Uri could be created from completely different strings: http address, ftp address, coordinates, phone numbers, contacts.</p>
<p>Content type could be identified by <strong>Scheme</strong>. Also, Scheme could be adjusted within <strong>Intent Filter</strong>, and we can filter Intents with Uri’s data type we need, only http for example. We will do it later, and now let’s code a simple example, where we will declare an Intent with action and data, send it and wait for result. Let’s view http address, coordinates on a map and open the dialer.</p>
<p>We need Google Maps App to view map coordinates. There is no such an app in standard android system images (that you downloaded with SDK Manager). We need system image which name begins with "Google APIs".</p>
<p> </p>
<p>Create AVD with platform Google APIs with API Level 10. Call it any way you want</p>
<p><img src="https://lh3.googleusercontent.com/-Y2wWS9RFCoc/TqRT2R6dmtI/AAAAAAAAAiA/hRpMbJOsvTo/s640/20111023_L0031_L_AVD.jpg" alt="" width="404" height="640" border="0"/></p>
<p> </p>
<p>Let’s create a project. Note, that we use Google APIs 2.3.3</p>
<p><img src="https://lh4.googleusercontent.com/-SqAm5XfrDCE/TqRGSl4h5iI/AAAAAAAAAhY/hDJs7eEBNB0/s800/20111023_L0031_L_Platform.jpg" alt="" width="525" height="573" border="0"/></p>
<p> </p>
<p><strong>Project name</strong>: P0311_SimpleIntents<br/> <strong>Build Target</strong>: Google APIs 2.3.3<br/> <strong>Application name</strong>: SimpleIntents<br/> <strong>Package name</strong>: ru.startandroid.develop.p0311simpleintents<br/> <strong>Create Activity</strong>: MainActivity</p>
<p><em>(If you didn’t manage to load Google APIs, just create a new project as usual with Android 2.3.3 platform. Google Maps just won’t work within this example). </em></p>
<p> </p>
<p>Let’s code<strong> main.xml</strong></p>
<pre class="brush:xml">&lt;?xml version="1.0" encoding="utf-8"?&gt;
&lt;LinearLayout
    xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="fill_parent"
    android:layout_height="fill_parent"
    android:orientation="horizontal"&gt;
        &lt;Button
            android:id="@+id/btnWeb"
            android:layout_width="match_parent"
            android:layout_height="wrap_content"
            android:layout_margin="10dp"
            android:layout_weight="1"
            android:text="Web"&gt;
        &lt;/Button&gt;
        &lt;Button
            android:id="@+id/btnMap"
            android:layout_width="match_parent"
            android:layout_height="wrap_content"
            android:layout_margin="10dp"
            android:layout_weight="1"
            android:text="Map"&gt;
        &lt;/Button&gt;
        &lt;Button
            android:id="@+id/btnCall"
            android:layout_width="match_parent"
            android:layout_height="wrap_content"
            android:layout_margin="10dp"
            android:layout_weight="1"
            android:text="Call"&gt;
        &lt;/Button&gt;
&lt;/LinearLayout&gt;</pre>
<p>There are three buttons on the screen: the first one will open a web-page, the second is for map opening and the third is for dialer. </p>
<p> </p>
<p>Let's code<strong> MainActivity.java</strong>:</p>
<pre class="brush:java">package ru.startandroid.develop.p0311simpleintents;

import android.app.Activity;
import android.content.Intent;
import android.net.Uri;
import android.os.Bundle;
import android.view.View;
import android.view.View.OnClickListener;
import android.widget.Button;

public class MainActivity extends Activity implements OnClickListener {

  Button btnWeb;
  Button btnMap;
  Button btnCall;

  /** Called when the activity is first created. */
  @Override
  public void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.main);

    btnWeb = (Button) findViewById(R.id.btnWeb);
    btnMap = (Button) findViewById(R.id.btnMap);
    btnCall = (Button) findViewById(R.id.btnCall);

    btnWeb.setOnClickListener(this);
    btnMap.setOnClickListener(this);
    btnCall.setOnClickListener(this);
  }

  @Override
  public void onClick(View v) {
    Intent intent;
    switch (v.getId()) {
    case R.id.btnWeb:
      intent = new Intent(Intent.ACTION_VIEW, Uri.parse("http://developer.android.com"));
      startActivity(intent);
      break;
    case R.id.btnMap:
      intent = new Intent();
      intent.setAction(Intent.ACTION_VIEW);
      intent.setData(Uri.parse("geo:55.754283,37.62002"));
      startActivity(intent);
      break;
    case R.id.btnCall:
      intent = new Intent(Intent.ACTION_DIAL);
      intent.setData(Uri.parse("tel:12345"));
      startActivity(intent);
      break;
    }
  }
}</pre>
<p>I followed three different ways to create an Intent with attributes.</p>
<p>I used <a href="http://developer.android.com/reference/android/content/Intent.html#Intent(java.lang.String, android.net.Uri)" target="_blank" rel="noopener noreferrer">Intent (String action, Uri uri)</a> constructor for <strong>btnWeb</strong>. It creates an <strong>intent</strong> with <strong>action</strong> and <strong>data</strong> variables. We use a standard action – ACTION_VIEW, it indicates if we want to <strong>view</strong> something. We pass Uri, created from web link http://developer.android.com, as <strong>data</strong> variable. If we try to describe in words our code, we will get something like this: our intent indicates if we want to view this link’s content and looking for activity to do it.</p>
<p>In case with <strong>btnMap</strong> I used <a href="http://developer.android.com/reference/android/content/Intent.html#Intent()" target="_blank" rel="noopener noreferrer">Intent()</a> constructor. It just creates <strong>intent</strong>, and we determine <strong>action</strong> and <strong>data</strong> attributes for it within next rows. <strong>action</strong> is ACTION_VIEW again and we use Uri was which created from coordinates - 55.754283,37.62002, as <strong>data</strong>. This intent indicates that we want to see the coordinates on a map.</p>
<p>We use <a href="http://developer.android.com/reference/android/content/Intent.html#Intent(java.lang.String)" target="_blank" rel="noopener noreferrer">Intent (String action)</a> constructor for <strong>btnCall</strong>. It needs an <strong>action</strong> to be set at once, but data attribute will be determined later. action (in this case it is ACTION_DIAL ) opens dialer and dial a number determined in <strong>data</strong>, but it doesn’t launch a call. We set Uri, created from phone number 12345, as data.</p>
<p>All this three ways lead us to same result: we will get an <strong>intent</strong> with determined action and data attributes. You decide which one to use, depending on the situation.</p>
<p>We need an internet connection to be on our PC, because App will use it to open the link and show the map.</p>
<p>Add a Users Permission element within app’s manifest and choose android.permission.INTERNET for it’s name. It will allow internet connection to the app. In my case it works even without this permission, but I can’t get why…</p>
<p> </p>
<p>Let’s save and launch the app.</p>
<p> </p>

<p>Standard browser will be launched, showing our link content.</p>
<p><img src="https://lh4.googleusercontent.com/-hprC7pbt_tE/TqRGS4VIICI/AAAAAAAAAhk/0ZauEBdDq18/s640/20111023_L0031_L_WebScreen.jpg" alt="" width="640" height="454" border="0"/></p>
<p> </p>
<p>Return to the main screen and click Map.</p>
<p>Map which shows place, according to determined coordinates, is now displayed.</p>
<p><img src="https://lh4.googleusercontent.com/--uDBALCUWoI/TqRGSaKiMcI/AAAAAAAAAhQ/tZ8jDgBnljM/s640/20111023_L0031_L_MapScreen.jpg" alt="" width="640" height="454" border="0"/></p>
<p> </p>
<p>Return to the main screen and click Call.</p>
<p>Standard dialer is displayed and we see that determined within data number is already dialed. We just need to click the call-button.</p>
<p><img src="https://lh4.googleusercontent.com/-VrHil4zQwsg/TqRGSKtf5dI/AAAAAAAAAhc/7FQHGVIRulk/s640/20111023_L0031_L_CallScreen.jpg" alt="" width="640" height="454" border="0"/></p>
<p> </p>
<p>Most likely, you may have lot’s of "what if" questions now. I can answer some of them at once and suggest you to make some experiments within this app.</p>
<p>1) What if we don’t use "geo" prefix:</p>
<p>System will warn you, that there is no suitable activity, because Map Activity is adjusted to work with Schema = geo, I think.</p>
<p>If we don’t use tel prefix, Call Activity similarly won’t work.</p>
<p> </p>
<p>2) What if we use geo with out of range coordinates:</p>
<p>If we use, out of range coordinates for example, like geo:a,b, map Activity will be launched, but will warn: "Unable to load the URL". I.e. data was suitable by Schema, but was incorrect.</p>
<p> </p>
<p>3) What if we use wright coordinates, but the action will be ACTION_EDIT instead of ACTION_VIEW.</p>
<p>It turns out that we want to edit place on map, defined with this coordinates. System will warn us, that there is no appropriate activity. Google app is waiting for Intent with action = ACTION_VIEW. It will show us place on a map, but it didn’t agree to edit this place )</p>
<p> </p>
<p>It’s necessary to understand, that all system apps are designed for specific actions with specific data. So, if you try to make a call to http link or find a phone number on a map, system just won’t find appropriate activity.</p>

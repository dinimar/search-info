	<section class="article-content clearfix">
		<p>В этом уроке мы:</p>
<p>- рисуем экран программно, а не через layout-файл</p>
 
<p>До этого мы создавали экран с помощью<strong> layout-файлов</strong>. Но то же самое мы можем делать и <strong>программно</strong>.</p>
<p>Создадим проект:</p>
<p> </p>
<p><strong>Project name</strong>: P0161_DynamicLayout<br/> <strong>Build Target</strong>: Android 2.3.3<br/> <strong>Application name</strong>: DynamicLayout<br/> <strong>Package name</strong>: ru.startandroid.develop.dinamiclayout<br/> <strong>Create Activity</strong>: MainActivity</p>
<p>Открываем <strong>MainActivity.java</strong> и обратим внимание на строку:</p>
<div class="java">
<pre class="brush:java">setContentView(R.layout.main);</pre>
<p>Напомню, что в этой строке мы указываем, что <strong>Activity </strong>в качестве экрана будет использовать <strong>layout-файл main.xml</strong>. Есть другая реализация этого метода, которая на вход принимает не layout-файл, а <strong>View</strong>-элемент и делает его корневым. В layout-файлах корневой элемент обычно <strong>LinearLayout</strong>, мы тоже используем его.</p>
</div>
<div class="java">
<pre class="brush:java">    @Override
    public void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        // создание LinearLayout
        LinearLayout linLayout = new LinearLayout(this);
        // установим вертикальную ориентацию
        linLayout.setOrientation(LinearLayout.VERTICAL);
        // создаем LayoutParams  
        LayoutParams linLayoutParam = new LayoutParams(LayoutParams.MATCH_PARENT, LayoutParams.MATCH_PARENT); 
        // устанавливаем linLayout как корневой элемент экрана 
        setContentView(linLayout, linLayoutParam);
    }</pre>
<p> </p>
</div>
<p>Обновим импорт – <strong>CTRL+SHIFT+O</strong>. Eclipse предложит нам выбрать, какой именно <strong>LayoutParams </strong>мы будем использовать. Тут надо остановиться подробнее. Вспомним теорию про экраны. Экран <a href="http://developer.android.com/guide/topics/ui/index.html#ViewHierarchy" target="_blank">состоит</a> из ViewGroup и вложенных в них View.</p>
<p><img style="border: 1px solid black;" src="https://lh4.googleusercontent.com/-F91ssebAJlo/Tm-ZlYN7LzI/AAAAAAAAAUE/noad5YNMnds/s800/20110913_L0016_L_ViewGroups.png" alt="" width="312" height="211" border="0"/></p>
<p>Известные нам примеры <strong>ViewGroup </strong>– это <strong>LinearLayout</strong>, <strong>TableLayout</strong>, <strong>RelativeLayout </strong>и т.д. Каждая из этих ViewGroup <a href="http://developer.android.com/guide/topics/ui/declaring-layout.html#layout-params" target="_blank">имеет вложенный класс LayoutParams</a>. Базовым для этих LayoutParams является <a href="http://developer.android.com/reference/android/view/ViewGroup.LayoutParams.html" target="_blank">ViewGroup.LayoutParams</a>.</p>
<p><strong>ViewGroup</strong>.<strong>LayoutParams </strong>имеет всего два атрибута: <strong>height </strong>и <strong>width</strong>. Его подкласс <a href="http://developer.android.com/reference/android/view/ViewGroup.MarginLayoutParams.html" target="_blank">ViewGroup.MarginLayoutParams</a> наследует два этих атрибута и имеет свои четыре: <strong>bottomMargin</strong>, <strong>leftMargin</strong>, <strong>rightMargin</strong>, <strong>topMargin</strong>. Класс <a href="http://developer.android.com/reference/android/widget/LinearLayout.LayoutParams.html" target="_blank">LinearLayout.LayoutParams</a> в свою очередь является подклассом ViewGroup.MarginLayoutParams, наследует от него уже 6 аттрибутов и добавляет свои два: <strong>gravity </strong>и <strong>weight</strong>.</p>
<p>Т.е. объект <strong>LinearLayout </strong>имеет вложенный класс <strong>LinearLayout.LayoutParams</strong> с layout-аттрибутами. И эти аттрибуты распространяются на все дочерние View и ViewGroup.</p>
<p><img style="border: 1px solid black;" src="https://lh4.googleusercontent.com/-xz2bRjH39vw/Tm-ZkRBvNMI/AAAAAAAAATw/ht4jwihDyJE/s800/20110913_L0016_L_LayoutParams.png" alt="" width="514" height="300" border="0"/></p>
<p> </p>
<p>Т.е. View, находящаяся в LinearLayout имеет один набор layout-параметров:</p>
<p><img style="border: 1px solid black;" src="https://lh4.googleusercontent.com/-ZPUWatnuYjY/Tm-ZkZZ7mQI/AAAAAAAAAT0/w9bZt3ar-MQ/s800/20110913_L0016_L_LinLayoutParams.jpg" alt="" width="313" height="282" border="0"/></p>
<p> </p>
<p>а View из RelativeLayout – другой:</p>
<p><img style="border: 1px solid black;" src="https://lh5.googleusercontent.com/-t_UkqqNzvVs/Tm-Zkp1rrdI/AAAAAAAAAT4/3R1oFlO6ZyU/s800/20110913_L0016_L_RelLayoutParams.jpg" alt="" width="328" height="543" border="0"/></p>
<p>Есть и общие элементы, т.к. родители у этих ViewGroup одни.</p>
<p> </p>
<p>Вернемся в Eclipse, он ждет нашего выбора. Используем базовый класс ViewGroup.LayoutParams</p>
<p><img src="https://lh3.googleusercontent.com/-n2e7EEZK-94/Tm-ZkUHe6kI/AAAAAAAAATs/iYryeP2PpLk/s800/20110913_L0016_L_LayoutParamsChoice.jpg" alt="" width="482" height="437" border="0"/></p>
<p> </p>
<p>Давайте разберем код. Мы создаем <strong>LinearLayout </strong>и ставим <strong>вертикальную </strong>ориентацию. Далее создаем <strong>LayoutParams</strong>. Конструктор на вход принимает два параметра: <strong>width </strong>и <strong>height</strong>. Мы оба ставим <strong>MATCH_PARENT</strong>. Далее вызывается метод <a href="http://developer.android.com/reference/android/app/Activity.html#setContentView(android.view.View, android.view.ViewGroup.LayoutParams)" target="_blank">setContentView</a>. На вход ему подается <strong>LinearLayout </strong>и <strong>LayoutParams</strong>. Это означает, что корневым элементом <strong>Activity </strong>будет <strong>LinearLayout </strong>с layout-свойствами из <strong>LayoutParams</strong>.</p>
<p>Если мы сейчас запустим приложение, то ничего не увидим, т.к. LinearLayout – прозрачен. Давайте добавлять в LinearLayout View-компоненты.</p>
<div class="java">
<pre class="brush:java">        LayoutParams lpView = new LayoutParams(LayoutParams.WRAP_CONTENT, LayoutParams.WRAP_CONTENT);
        
        TextView tv = new TextView(this);
        tv.setText("TextView");
        tv.setLayoutParams(lpView);
        linLayout.addView(tv);
        
        Button btn = new Button(this);
        btn.setText("Button");
        linLayout.addView(btn, lpView);</pre>
<p>Мы снова создаем объект <strong>LayoutParams </strong>с атрибутами <strong>width </strong>= <strong>wrap_content </strong>и <strong>height </strong>= <strong>wrap_content</strong>. Теперь если мы присвоим этот объект какому-либо View, то это <strong>View </strong>будет иметь <strong>ширину </strong>и <strong>высоту по содержимому</strong>.</p>
</div>
<p>Далее мы создаем <strong>TextView</strong>, настраиваем его текст, присваиваем ему выше созданный LayoutParams и добавляем в LinearLayout с помощью метода <a href="http://developer.android.com/reference/android/view/ViewGroup.html#addView(android.view.View)" target="_blank">addView (View child)</a>.</p>
<p>С <strong>Button </strong>аналогично – создаем, правим текст, а затем используем другую реализацию метода <a href="http://developer.android.com/reference/android/view/ViewGroup.html#addView(android.view.View, android.view.ViewGroup.LayoutParams)" target="_blank">addView (View child, ViewGroup.LayoutParams params)</a>, которая одновременно добавляет <strong>Button </strong>в <strong>LinearLayout </strong>и присваивает для Button указанный <strong>LayoutParams</strong>. Результат будет тот же, что и с TextView, но вместо двух строк кода получилась одна.</p>
<p>Обратите внимание, что для<strong> двух объектов View</strong> я использовал <strong>один объект LayoutParams</strong> - lpView. Оба View-объекта считают параметры из LayoutParams и используют их.</p>
<p>Сохраним и запустим приложение. Видим, что компоненты на экране появились. И видно, что их высота и ширина определена по содержимому (wrap_content).</p>
<p><img src="https://lh5.googleusercontent.com/-zvNtPE8WS7o/Tm-ZkxSyJFI/AAAAAAAAAT8/65P1OwPHqvg/s640/20110913_L0016_L_Screen1.jpg" alt="" width="640" height="454" border="0"/></p>
<p> </p>
<p>Объект <strong>lpView </strong>имеет базовый тип <strong>android.view.ViewGroup.LayoutParams</strong>. А значит позволит настроить только ширину и высоту. Но для <strong>View </strong>в <strong>LinearLayout </strong>доступны, например, отступ слева или выравнивание по правому краю. И если мы хотим их задействовать, значит надо использовать <strong>LinearLayout.LayoutParams</strong>:</p>
<div class="java">
<pre class="brush:java">        LinearLayout.LayoutParams leftMarginParams = new LinearLayout.LayoutParams(
                LayoutParams.WRAP_CONTENT, LayoutParams.WRAP_CONTENT);
        leftMarginParams.leftMargin = 50;
        
        Button btn1 = new Button(this);
        btn1.setText("Button1");
        linLayout.addView(btn1, leftMarginParams);</pre>
<p>Смотрим код. Мы создаем объект типа<strong> LinearLayout.LayoutParams</strong> с помощью такого же конструктора, как и для обычного LayoutParams, указывая <strong>width </strong>и <strong>height</strong>. Затем мы указываем <strong>отступ слева</strong> = 50. Отступ здесь указывается в <strong>пикселах</strong>. Далее схема та же: создаем объект, настраиваем текст и добавляем его в LinearLayout c присвоением LayoutParams.</p>
</div>
<p>Аналогично добавим компонент с выравниванием:</p>
<div class="java">
<pre class="brush:java">        LinearLayout.LayoutParams rightGravityParams = new LinearLayout.LayoutParams(
                LayoutParams.WRAP_CONTENT, LayoutParams.WRAP_CONTENT);
        rightGravityParams.gravity = Gravity.RIGHT;
        
        Button btn2 = new Button(this);
        btn2.setText("Button2");
        linLayout.addView(btn2, rightGravityParams);</pre>
<p> </p>
<p>Сохраним и запустим. Button1 имеет отступ 50px. А Button2 выравнена по правому краю:</p>
</div>
<p><img src="https://lh3.googleusercontent.com/-lSvGCW3-e_E/Tm-ZlMnnrjI/AAAAAAAAAUA/N5iTXMiYLqU/s640/20110913_L0016_L_Screen2.jpg" alt="" width="640" height="454" border="0"/></p>
<p> </p>
<p> </p>
<p>Вероятно, эта тема будет не очень понятна с первого раза. Поэтому на следующих двух уроках мы закрепим эти знания и попрактикуемся в добавлении элементов на экран и их настройке.</p>
<p>Полный код урока:</p>
<div class="java">
<pre class="brush:java">public class MainActivity extends Activity {
    /** Called when the activity is first created. */
    @Override
    public void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        // создание LinearLayout
        LinearLayout linLayout = new LinearLayout(this);
        // установим вертикальную ориентацию
        linLayout.setOrientation(LinearLayout.VERTICAL);
        // создаем LayoutParams  
        LayoutParams linLayoutParam = new LayoutParams(LayoutParams.MATCH_PARENT, LayoutParams.MATCH_PARENT); 
        // устанавливаем linLayout как корневой элемент экрана 
        setContentView(linLayout, linLayoutParam);
        
        LayoutParams lpView = new LayoutParams(LayoutParams.WRAP_CONTENT, LayoutParams.WRAP_CONTENT);
        
        TextView tv = new TextView(this);
        tv.setText("TextView");
        tv.setLayoutParams(lpView);
        linLayout.addView(tv);
        
        Button btn = new Button(this);
        btn.setText("Button");
        linLayout.addView(btn, lpView);

        
        LinearLayout.LayoutParams leftMarginParams = new LinearLayout.LayoutParams(
                LayoutParams.WRAP_CONTENT, LayoutParams.WRAP_CONTENT);
        leftMarginParams.leftMargin = 50;
        
        Button btn1 = new Button(this);
        btn1.setText("Button1");
        linLayout.addView(btn1, leftMarginParams);

        
        LinearLayout.LayoutParams rightGravityParams = new LinearLayout.LayoutParams(
                LayoutParams.WRAP_CONTENT, LayoutParams.WRAP_CONTENT);
        rightGravityParams.gravity = Gravity.RIGHT;
        
        Button btn2 = new Button(this);
        btn2.setText("Button2");
        linLayout.addView(btn2, rightGravityParams);
    }
}</pre>
<p> </p>
</div>
<p>На следующем уроке:</p>
<p>- добавляем компоненты на экран во время работы приложения</p><div><hr>

	<section class="article-content clearfix">
		<p>This lesson:</p>
<p>- using ListView to create a List.</p>
 
<p> </p>
<p>Before we begin to study <strong>ListView, </strong>I suggest to recall the last lesson and the mechanism of the list building, which we used.  We went through an array of data, created a list item within every iteration, filled it with data and added to the list.</p>
<p>The <strong>adapter </strong>will create list items for us while <strong>ListView </strong>creating. The <strong>adapter </strong>needs data and a layout-resource for list item. After passing arguments to the adapter, we assign the <strong>adapter </strong>to the <strong>ListView </strong>list. While building, the list requests the items from the adapter, the adapter creates them (using data and layout) and returns them to the list. As a result we see the finished list.</p>
<p>There are different types of lists and adapters. We will study the most simple variant so far.</p>
<p> </p>
<p>Let’s create a project: </p>
<p><b>Project name</b>: P0421_SimpleList<br/><b>Build Target</b>: Android 2.3.3<br/><b>Application name</b>: SimpleList<br/><b>Package name</b>: ru.startandroid.develop.p0421simplelist<br/><b>Create Activity</b>: MainActivity</p>
<p>Open <strong>main.xml</strong> and add <strong>ListView </strong>to the layout (Composite tab):</p>
<pre class="brush:xml">&lt;?xml version="1.0" encoding="utf-8"?&gt;
&lt;LinearLayout
    xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="fill_parent"
    android:layout_height="fill_parent"
    android:orientation="vertical"&gt;
    &lt;TextView
        android:layout_width="fill_parent"
        android:layout_height="wrap_content"
        android:text="@string/hello"&gt;
    &lt;/TextView&gt;
    &lt;ListView
        android:id="@+id/lvMain"
        android:layout_width="match_parent"
        android:layout_height="wrap_content"&gt;
    &lt;/ListView&gt;
&lt;/LinearLayout&gt;<br/><br/></pre>
<p> </p>
<p><strong>ListView </strong>is the list component.</p>
<p> </p>
<p>Now, we need to create an <strong>adapter</strong>. Open  <strong>MainActivity.java</strong> and code: </p>
<p> </p>
<div class="java">
<pre class="brush:java">package ru.startandroid.develop.p0421simplelist;

import android.app.Activity;
import android.os.Bundle;
import android.widget.ArrayAdapter;
import android.widget.ListView;

public class MainActivity extends Activity {

  String[] names = { "Иван", "Марья", "Петр", "Антон", "Даша", "Борис",
      "Костя", "Игорь", "Анна", "Денис", "Андрей" };

  /** Called when the activity is first created. */
  public void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.main);

    // находим список
    ListView lvMain = (ListView) findViewById(R.id.lvMain);

    // создаем адаптер
    ArrayAdapter&lt;String&gt; adapter = new ArrayAdapter&lt;String&gt;(this,
        android.R.layout.simple_list_item_1, names);

    // присваиваем адаптер списку
    lvMain.setAdapter(adapter);

  }
}</pre>
</div>
<p>You won’t believe, but this is all code we need for the list creation )</p>
<p>We use array of names as data. Within the onCreate method we determine the list, create an adapter and set it for the list. Let’s find out how we created the adapter.</p>
<p>We used this constructor:  <a href="http://developer.android.com/reference/android/widget/ArrayAdapter.html#ArrayAdapter(android.content.Context,%20int,%20T%5B%5D)">public ArrayAdapter (Context context, int textViewResourceId, T[] objects) </a></p>
<p>and passed the next parameters to it:<br/><strong>this </strong>(context);<br/><strong>android.R.layout.simple_list_item_1</strong> b is a system layout file, which contains the textView.<br/><strong>names </strong>is the data array we want to display;</p>
<p> </p>
<p>We can look at content of  <b>simple_list_item_1</b>. Find <b>Android 2.3.3</b>. in your project, expand it, and expand <b>android.jar</b></p>
<p><img src="https://lh5.googleusercontent.com/-V1_HuRy240Y/Tts5Ce8MZ2I/AAAAAAAAArA/yCrsiYkh6Fc/s800/20111203_L0042_L_android.jpg" alt=""/></p>
<p> </p>
<p>Scroll to the bottom and open <strong>res.layout</strong>.</p>
<p><img src="https://lh4.googleusercontent.com/-ODanktnvKjA/Tts5Cs8mXZI/AAAAAAAAArE/F5vtyioF48o/s800/20111203_L0042_L_reslayout.jpg" alt="" width="233" height="99"/></p>
<p> </p>
<p>And inside we can find <strong>simple_list_item_1</strong>, which we use</p>
<p><img src="https://lh5.googleusercontent.com/-tpEASEpzh3c/Tts5C0zSaSI/AAAAAAAAArI/H4S6sZSyLhE/s800/20111203_L0042_L_simplelistitem1.jpg" alt="" width="220" height="144"/></p>
<p> </p>
<p>Make a double click on it to see the contents:</p>
<pre class="brush:xml">&lt;?xml version="1.0" encoding="utf-8"?&gt;
&lt;TextView
    xmlns:android="http://schemas.android.com/apk/res/android"
    android:id="@android:id/text1"
    android:layout_width="match_parent"
    android:layout_height="wrap_content"
    android:textAppearance="?android:attr/textAppearanceLarge"
    android:gravity="center_vertical"
    android:paddingLeft="6dip"
    android:minHeight="?android:attr/listPreferredItemHeight"&gt;
&lt;/TextView&gt;</pre>
<p>This is a simple text view with a set of parameters.</p>
<p> </p>
<p>When the list asks for the next item while building, the adapter takes this Layout-resource <strong>simple_list_item_1, </strong>pass it through <strong>LayoutInflater</strong> and gets the <strong>View</strong>, converts <strong>View </strong>to TextView, set its text with data from the data array and returns it to the list.</p>
<p>Let’s save and launch. We can see the list of our data.</p>
<p> </p>
<p><img src="https://lh3.googleusercontent.com/-I3R_9GJOfV8/Tts5Dh7pl3I/AAAAAAAAArU/z1ewAn3cyO0/s640/20111203_L0042_L_system.jpg" alt="" width="640" height="454"/></p>
<p>Using the system layout file simple_list_item_1 is good, because we don’t need to code it by ourselves. But we can create our own resource, in case the system’s resource doesn’t fit us.   </p>
<p>Let’s create layout file  <strong>my_list_item.xml</strong> in our project’s res/layout folder:</p>
<pre class="brush:xml">&lt;?xml version="1.0" encoding="utf-8"?&gt;
&lt;TextView
    xmlns:android="http://schemas.android.com/apk/res/android"
    android:id="@+id/textView1"
    android:layout_width="match_parent"
    android:layout_height="wrap_content"
    android:gravity="center_horizontal"
    android:padding="5dp"
    android:text="TextView"
    android:textColor="#00FF00"
    android:textSize="24sp"&gt;
&lt;/TextView&gt;</pre>
<p><strong>TextView </strong>with specified color, text size, alignment and padding</p>
<p> </p>
<p>Let’s change the code and pass <strong>my_list_item</strong> layout resource we created to the adapter:</p>
<div class="java">
<pre class="brush:java; highlight: [3]">    // создаем адаптер
    ArrayAdapter&lt;String&gt; adapter = new ArrayAdapter&lt;String&gt;(this,
        R.layout.my_list_item, 
        names);</pre>
<p>Now the adapter will use it for list items building.</p>
<p> </p>
<p>Let’s save and launch the app. We will see our list colored with green.</p>
</div>
<p><img src="https://lh5.googleusercontent.com/-nX15KpJnxqE/Tts5CHhBZ_I/AAAAAAAAAq8/ziQ-6r6Dwvc/s640/20111203_L0042_L_my.jpg" alt="" width="640" height="454"/></p>
<p>Within the layout resource you may use any View class’ child instead of TextView, it could be Button for example. The main thing is for the object to undergo conversion to TextView. The adapter will specify its text with the setText method and return it to the list.</p>
<p> </p>
<p> </p>
<h3>A little bit about Context</h3>
<p>In one of the previous lessons  I told that <strong>Context </strong>is used to get access to the <strong>basic </strong>application functionality. In this lesson we got a good confirmation for these words.</p>
<p>ArrаyAdapter uses LayoutInflater to convert layout-resource into View. But getting LayoutInflater is one of the <strong>basic </strong>functions and it is <strong>unavailable </strong>for  <strong>Arr</strong><strong>а</strong><strong>yAdapter </strong>class. That’s why we pass a link to Activity as a context into ArrаyAdapter (Activity has an access to the basic functions through the ascending hierarchy of classes). And the ArrayAdapter class <strong>uses </strong>the <strong>context </strong>that was passed to it, to get LayoutInflater. It couldn’t do it without the context.</p>
<p> </p>

	<section class="article-content clearfix">
		<p>В этом уроке:</p>
<p>- используем транзакции при работе с БД</p>
 
<p> </p>
<p>Что такое БД-<strong>транзакция</strong>, думаю объяснять особо не надо. Она используется при работе с данными по принципу «<strong>все или ничего</strong>». Т.е., например, вам нужно вставить пачку данных. Но вставить надо так, чтобы или все вставилось или ничего не вставилось.  И если в процессе половина записей прошла, а другая нет – должна быть возможность <strong>откатить </strong>изменения.</p>
<p>Напишем простое приложение и исследуем возможности <strong>SQLite </strong>в этом плане.</p>
<p>Создадим проект:</p>
<p><strong>Project name</strong>: P0381_SQLiteTransaction<br/> <strong>Build Target</strong>: Android 2.3.3<br/> <strong>Application name</strong>: SQLiteTransaction<br/> <strong>Package name</strong>: ru.startandroid.develop.p0381sqlitetransaction<br/> <strong>Create Activity</strong>: MainActivity</p>
<p> </p>
<p>Открываем<strong> MainActivity.java</strong> и пишем:</p>
<pre class="brush:java">package ru.startandroid.develop.p0381sqlitetransaction;

import android.app.Activity;
import android.content.ContentValues;
import android.content.Context;
import android.database.Cursor;
import android.database.sqlite.SQLiteDatabase;
import android.database.sqlite.SQLiteOpenHelper;
import android.os.Bundle;
import android.util.Log;

public class MainActivity extends Activity {

  final String LOG_TAG = "myLogs";

  DBHelper dbh;
  SQLiteDatabase db;

  /** Called when the activity is first created. */
  public void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.main);
    Log.d(LOG_TAG, "--- onCreate Activity ---");
    dbh = new DBHelper(this);
    myActions();
  }

  void myActions() {
    db = dbh.getWritableDatabase();
    delete(db, "mytable");
    insert(db, "mytable", "val1");
    read(db, "mytable");
    dbh.close();
  }

  void insert(SQLiteDatabase db, String table, String value) {
    Log.d(LOG_TAG, "Insert in table " + table + " value = " + value);
    ContentValues cv = new ContentValues();
    cv.put("val", value);
    db.insert(table, null, cv);
  }

  void read(SQLiteDatabase db, String table) {
    Log.d(LOG_TAG, "Read table " + table);
    Cursor c = db.query(table, null, null, null, null, null, null);
    if (c != null) {
      Log.d(LOG_TAG, "Records count = " + c.getCount());
      if (c.moveToFirst()) {
        do {
          Log.d(LOG_TAG, c.getString(c.getColumnIndex("val")));
        } while (c.moveToNext());
      }
      c.close();
    }
  }

  void delete(SQLiteDatabase db, String table) {
    Log.d(LOG_TAG, "Delete all from table " + table);
    db.delete(table, null, null);
  }

  // класс для работы с БД
  class DBHelper extends SQLiteOpenHelper {

    public DBHelper(Context context) {
      super(context, "myDB", null, 1);
    }

    public void onCreate(SQLiteDatabase db) {
      Log.d(LOG_TAG, "--- onCreate database ---");

      db.execSQL("create table mytable ("
          + "id integer primary key autoincrement," 
          + "val text"
          + ");");
    }

    public void onUpgrade(SQLiteDatabase db, int oldVersion, int newVersion) {

    }
  }

}</pre>
<p> </p>
<p>Разбираем код. Я создал несколько методов, где сгруппировал операции: <strong>insert </strong>– набор операций для вставки записи, <strong>read </strong>– чтение всех записей, <strong>delete </strong>– удаление всех записей. Класс <strong>DBHelper </strong>– для управления БД. Интересовать нас будет метод <strong>myActions</strong>. Сейчас в нем мы подключаемся к БД, очищаем таблицу mytable, вставляем строку с значением <em>val1</em>, выводим в лог все записи из таблицы и отключаемся.</p>
<p>Все сохраним, запустим приложение. Смотрим лог:</p>
<p><em>--- onCreate Activity ---<br/> --- onCreate database ---<br/> Delete all from table mytable<br/> Insert in table mytable value = val1<br/> Read table mytable<br/> Records count = 1<br/> val1</em></p>
<p>Все верно, запись вставилась и отобразилась.</p>
<p>Теперь попробуем использовать транзакцию. Поправим код <strong>myActions</strong> на этот:</p>
<pre class="brush:java">  void myActions() {
    db = dbh.getWritableDatabase();
    delete(db, "mytable");
    db.beginTransaction();
    insert(db, "mytable", "val1");
    db.endTransaction();
    insert(db, "mytable", "val2");
    read(db, "mytable");
    dbh.close();
  }</pre>
<p>Мы подключаемся к базе, чистим таблицу, открываем транзакцию методом <a href="http://developer.android.com/reference/android/database/sqlite/SQLiteDatabase.html#beginTransaction()" target="_blank">beginTransaction</a>, вставляем <em>val1</em>, закрываем транзакцию методом <a href="http://developer.android.com/reference/android/database/sqlite/SQLiteDatabase.html#endTransaction()" target="_blank">endTransaction</a>, вставляем <em>val2</em>, выводим содержимое в лог и отключаемся. Все сохраняем, запускаем и смотрим лог:</p>
<p><em>--- onCreate Activity ---<br/> Delete all from table mytable<br/> Insert in table mytable value = val1<br/> Insert in table mytable value = val2<br/> Read table mytable<br/> Records count = 1<br/> val2</em></p>
<p>По логу видно, что вставляли мы <strong>две </strong>записи, но прошла только <strong>вторая</strong>. Та, которая была в <strong>транзакции </strong>– не записалась. Это произошло потому, что мы явно не указали, что транзакция должна быть успешно <strong>закрыта</strong>. Если этого не сделать, то при закрытии транзакции все операции <strong>отменятся</strong>. Давайте исправимся. Снова перепишем <strong>myActions</strong>:</p>
<div class="java">
<pre class="brush:java">  void myActions() {
    db = dbh.getWritableDatabase();
    delete(db, "mytable");
    db.beginTransaction();
    insert(db, "mytable", "val1");
    db.setTransactionSuccessful();
    insert(db, "mytable", "val2");
    db.endTransaction();
    insert(db, "mytable", "val3");
    read(db, "mytable");
    dbh.close();
  }</pre>
<p> </p>
</div>
<p>Подключаемся к БД, чистим таблицу, открываем транзакцию, вставляем <em>val1</em>, подтверждаем успешность транзакции методом <a href="http://developer.android.com/reference/android/database/sqlite/SQLiteDatabase.html#setTransactionSuccessful()" target="_blank">setTransactionSuccessful</a>, вставляем <em>val2</em>, закрываем транзакцию, вставляем <em>val3</em>, выводим содержимое и отключаемся.</p>
<p>Сохраняем, запускаем, смотрим лог:</p>
<p><em>--- onCreate Activity ---<br/> Delete all from table mytable<br/> Insert in table mytable value = val1<br/> Insert in table mytable value = val2<br/> Insert in table mytable value = val3<br/> Read table mytable<br/> Records count = 3<br/> val1<br/> val2<br/> val3</em></p>
<p>Вставились все три записи. Обратите внимание - несмотря на то, что <em>val2 </em>мы вставляли уже после подтверждения успешности транзакции, запись вставилась, вошла в эту транзакцию. Но проводить операции после подтверждения транзакции не рекомендуется хелпом. </p>
<p> </p>
<p><strong>Транзакция </strong>при открытии ставит <strong>блокировку </strong>на базу. Убедимся в этом, попробуем создать новое подключение к БД во время транзакции. Перепишем <strong>myActions</strong>:</p>
<pre class="brush:java">  void myActions() {
    try {
      db = dbh.getWritableDatabase();
      delete(db, "mytable");

      db.beginTransaction();
      insert(db, "mytable", "val1");

      Log.d(LOG_TAG, "create DBHelper");
      DBHelper dbh2 = new DBHelper(this);
      Log.d(LOG_TAG, "get db");
      SQLiteDatabase db2 = dbh2.getWritableDatabase();
      read(db2, "mytable");
      dbh2.close();

      db.setTransactionSuccessful();
      db.endTransaction();

      read(db, "mytable");
      dbh.close();

    } catch (Exception ex) {
      Log.d(LOG_TAG, ex.getClass() + " error: " + ex.getMessage());
    }
  }</pre>
<p> </p>
<p>Подключаемся к базе, чистим таблицу, открываем транзакцию, вставляем запись, создаем новое подключение к БД - <strong>db2</strong>, читаем содержимое вторым подключением, закрываем второе подключение, успешно закрываем транзакцию, читаем содержимое первым подключением, закрываем первое подключение.</p>
<p>Все сохраним и запустим. Смотрим лог:</p>
<p><em>--- onCreate Activity ---<br/> Delete all from table mytable<br/> Insert in table mytable value = val1<br/> create DBHelper<br/> get db<br/> class android.database.sqlite.SQLiteException error: database is locked</em></p>
<p>Мы видим, что при попытке создать второе подключение к базе произошла ошибка <strong>SQLiteException </strong>– база <strong>заблокирована </strong>открытой транзакцией. Если вы теперь закоментите или удалите строки управления транзакцией и снова выполните код, то все пройдет успешно, т.к. никаких блокировок не будет.</p>
<p> </p>
<p>Наверняка есть некоторые вопросы по этой теме. Попробую здесь же ответить на некоторые.</p>
<h3>close</h3>
<p>Метод <strong>close </strong>есть и у <strong>SQLiteDatabase </strong>и у <strong>SQLiteOpenHelper</strong>. Какая между ними разница? Каким из них пользоваться для закрытия подключения?</p>
<p>Тут надо понимать один момент – объект <strong>SQLiteOpenHelper </strong>всегда предоставляет только <strong>одно </strong>подключение. Попробую объяснить этот механизм. У объекта <strong>SQLiteOpenHelper </strong>есть внутренний атрибут <strong>mDatabase </strong>типа <strong>SQLiteDatabase</strong>. Когда мы вызываем метод <strong>getWritableDatabase</strong>, объект <strong>SQLiteOpenHelper </strong>проверяет: если <strong>mDatabase </strong>не null и не закрыт, то он и идет в качестве <strong>return</strong>. Иначе  <strong>SQLiteOpenHelper </strong>выполняет подключение к БД, записывает новый <strong>SQLiteDatabase</strong>-объект в <strong>mDatabase </strong>и возвращает нам его. Т.е. метод <strong>getWritableDatabase </strong>либо возвращает <strong>существующее подключение </strong>к БД, либо создает новое в случае отсутствия подключения. Когда же выполняется метод <strong>close </strong>для <strong>SQLiteOpenHelper</strong>, то происходит вызов <strong>close </strong>для <strong>mDatabase </strong>и выполняется код <strong>mDatabase = null</strong>.</p>
<p>Рассмотрим на примере. Снова меняем метод myActions:</p>
<div class="java">
<pre class="brush:java">  void myActions() {
    db = dbh.getWritableDatabase();
    SQLiteDatabase db2 = dbh.getWritableDatabase();
    Log.d(LOG_TAG, "db = db2 - " + db.equals(db2));
    Log.d(LOG_TAG, "db open - " + db.isOpen() + ", db2 open - " + db2.isOpen());
    db2.close();
    Log.d(LOG_TAG, "db open - " + db.isOpen() + ", db2 open - " + db2.isOpen());
  }</pre>
<p>Сначала мы получаем <strong>db</strong>. При этом <strong>dbh </strong>проверяет свой внутренний атрибут <strong>mDatabase</strong>. Т.к. это первая попытка подключения, то <strong>mDatabase </strong>пуст, поэтому внутри <strong>dbh </strong>производится подключение и в <strong>mDatabase </strong>записывается свежесозданный <strong>SQLiteDatabase</strong>, и он же и возвращается в <strong>db </strong>из метода <strong>getWritableDatabse </strong>.</p>
</div>
<p>Затем мы из того же <strong>dbh </strong>получаем <strong>db2</strong>. <strong>dbh </strong>снова проверяет свой внутренний <strong>mDatabse</strong>, видит, что он уже <strong>не null и не закрыт</strong>, и возвращает нам его в наш <strong>db2</strong>. В итоге <strong>db </strong>и <strong>db2 </strong>равны и ссылаются на один и тот же объект. Проверяем это с помощью метода <strong>equals</strong>. Далее проверим, что <strong>db </strong>и <strong>db2 открыты</strong>. Потом <strong>закроем </strong>только <strong>db2</strong>, и еще раз проверим на <strong>открытость оба </strong>объекта.</p>
<p>Сохраняем, запускаем, смотрим лог:</p>
<p><em>--- onCreate Activity ---<br/> db = db2 – true<br/> db open - true, db2 open – true<br/> db open - false, db2 open - false</em></p>
<p>Видим, что <strong>equals </strong>вернул <strong>true</strong>. Затем видно, что <strong>db</strong> и <strong>db2 открыты</strong>. А после закрытия <strong>db2 </strong>видим, что закрыты оба объекта. Все оттого, что «оба объекта» – это всего лишь <strong>две ссылки </strong>на <strong>один объект</strong>.</p>
<p> </p>
<p>Если в коде вместо <strong>db2.close()</strong> поставить<strong> dbh.close()</strong> - эффект будет тот же. <strong>dbh </strong>вызовет метод <strong>close </strong>для <strong>mDatabase </strong>и обнулит его -<strong> mDatabase = null</strong>. А <strong>db </strong>и <strong>db2 </strong>будут ссылаться на закрытый <strong>SQLiteDatabase</strong>.</p>
<p>Я думаю, что правильнее вызывать <strong>close </strong>для <strong>SQLiteOpenHelper</strong>, а не для <strong>SQLiteDatabase</strong>. Т.к. гарантировано <strong>закрывается </strong>текущее открытое соединение и <strong>обнуляется </strong>внутренняя ссылка на объект.</p>
<p>Если вам надо получить второе открытое подключение к БД, то надо создавать новый экземпляр <strong>DBHelper </strong>и вызывать <strong>getWritableDatabase</strong>. Мы так делали чуть выше в примере с блокировкой транзакции.</p>
<p> </p>
<h3>read write</h3>
<p>В чем разница между <strong>getWritableDatabase </strong>и <strong>getReadableDatabase</strong>? Судя по хелпу, в обычной ситуации <strong>оба </strong>метода возвращают<strong> одно и то же</strong>. И оба позволят <strong>читать </strong>и <strong>менять </strong>БД. В случае же, например, проблемы<strong> отсутствия свободного места</strong> на устройстве, метод <strong>getReadableDatabase </strong>вернет БД только <strong>для чтения</strong>, а <strong>getWritableDatabase </strong>выдаст <strong>ошибку</strong>.</p>
<p> </p>
<h3>_id, как имя поля-идентификатора</h3>
<p>В различных источниках при работе с БД в качестве наименования поля-идентификатора в таблице используют не просто id, а _id. Почему?</p>
<p>Ответ нашелся в доках по Cursor-адаптерам. Цитата: "The Cursor must include a column named "_id" or this class will not work.". Т.е. если вы планируете использовать Cursor-адаптеры, то необходимо, чтобы таблица содержала поле _id, иначе адаптер не будет работать. </p>
<p> </p>
<h3>Блокировка</h3>
<p>Метод открытия транзакции <strong>beginTransaction </strong>ставит блокировку в режиме <strong>EXCLUSIVE</strong>. Т.е. БД блокируется и на чтение и на запись для других подключений. В SDK Android версии старше 2.3.3 появился метод <strong>beginTransactionNonExclusive</strong>, который ставит блокировку в режиме <strong>IMMEDIATE</strong>. Я подозреваю, что это позволит читать данные другим подключениям.</p>
<p>Если есть желание подробнее погрузиться в тему, вам <a href="http://www.enterra.ru/blog/android_issues_with_sqlite/" target="_blank">сюда</a>.</p>
<p> </p>
<h3>Синтаксис</h3>
<p>И кстати, рекомендуемая форма для использования транзакций такая:</p>
<div class="java">
<pre class="brush:java">    db.beginTransaction();
    try {
      ...
      db.setTransactionSuccessful();
    } finally {
      db.endTransaction();
    }</pre>
<p> </p>
</div>
<p>Это очень важно! Т.е. если вы <strong>открыли </strong>транзакцию, <strong>выполнили </strong>какие-либо действия и<strong> не закрыли</strong> транзакцию, то все операции будут считаться <strong>неуспешными </strong>и изменения не будут внесены в БД. Поэтому <strong>закрытие</strong> транзакции <strong>необходимо </strong>выполнять и <strong>finally </strong>нам это гарантирует.</p>
<p> </p>
<p>На следующем уроке:</p>
<p>- меняем версию и обновляем структуру БД в onUpgrade</p>

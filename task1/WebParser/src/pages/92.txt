	<section class="article-content clearfix">
		<p>Positioning of View-elements on the screen depends on the <b style="line-height: 1.3em;">ViewGroup</b> (layout), inside which they are located. In this lesson we will observe main types of <b>Layouts</b>.</p>
 
<p><i>Translated by Taras Leskiv (<a href="http://android-by-example.blogspot.com/" target="_blank" rel="noopener noreferrer">http://android-by-example.blogspot.com/</a>)</i></p>
<p><b>LinearLayout</b> - displays View-elements as a single row (if it is Horizontal) or a single column (if it is Vertical). I used it in the previous lesson, when demonstrating usage of layout-files during screen orientation change.</p>
<p><b>TableLayout</b> - displays elements in the form of a table, with rows and columns.</p>
<p><b>RelativeLayout</b> - each element’s position is configured relatively to other elements.</p>
<p><b>AbsoluteLayout</b> - each element is specified an absolute position on the screen in the coordinate system (x, y)</p>
<p>Lets observe these types</p>
<p> </p>
<h3>LinearLayout (LL)</h3>
<p>This type of a ViewGroup is set as default when creating new layout-files. It is really convenient and flexible enough to create screens of different complexity. LL has an Orientation property, which defines how will the child elements be positioned - in a vertical or horizontal line.</p>
<p>Let’s make a simple and clear example.</p>
<p>Create a project:</p>
<p><strong>Project name</strong>: P0061_Layouts<br/><strong>Build Target</strong>: Android 2.3.3<br/><strong>Application name</strong>: Layouts<br/><strong>Package name</strong>: ru.startandroid.develop.layouts<br/><strong>Create Activity</strong>: MainActivity</p>
<p>Open <b>main.xml</b> layout-file and place the following code inside:</p>
<pre class="brush:xml">&lt;?xml version="1.0" encoding="utf-8"?&gt;
&lt;LinearLayout
	xmlns:android="http://schemas.android.com/apk/res/android"
	xmlns:tools="http://schemas.android.com/tools"
	android:layout_width="match_parent"
	android:layout_height="match_parent"
	android:orientation="vertical"&gt;
&lt;/LinearLayout&gt;</pre>
<p>Now we have a LinearLayout with vertical orientation as a root element.</p>
<p>Drag three buttons from the left to the root LinearLayout. They are positioned horizontally.</p>
<p><img style="line-height: 1.3em; border: 1px solid black;" src="/images/stories/lessons/L0006/xL0006_005.jpg.pagespeed.ic.NPVC21uJ8u.jpg" alt="" width="516" height="309"/></p>
<p> </p>
<p>Now change <b>Orientation</b> property to <b>horizontal</b> for LL in Properties and save (CTRL+SHIFT+S) - buttons are positioned horizontally</p>
<p><img style="border: 1px solid black;" src="/images/stories/lessons/L0006/640x252xL0006_006.jpg.pagespeed.ic.FPTY31v6l7.jpg" alt="" width="640" height="252"/></p>
<p> </p>
<p>A ViewGroup can be inserted into another ViewGroup. Let’s place two other LL into one. Delete all the elements from main.xml (three buttons) except of the root LL. Specify vertical orientation for the root LL and add two new horizontal LL inside it. They are located in the Layouts section in the list of elements on the left. Let me remind you, that you can drag and drop elements from the list not only to the screen, but also to the specific element in the Outline tab.</p>
<p>Add three buttons inside each horizontal LL. Now we have got two horizontals rows of buttons.</p>
<p><img style="border: 1px solid black;" src="/images/stories/lessons/L0006/xL0006_010.jpg.pagespeed.ic.1_Hl4F8JgY.jpg" alt="" width="562" height="311"/></p>
<p> </p>
<h3>TableLayout (TL)</h3>
<p>TableLayout consists of rows <b>TableRow</b> (TR). And each TR contains View elements that form columns. So the number of Views in TR is a number of columns. But the number of columns in  a table must be equal for all rows. That’s why, when different TRs have different numbers of View-elements (columns), the overall number of columns is defined by the TR with the maximum number. Let’s observe an example.</p>
<p>Create layout-file <b>tlayout.xml</b> with TableLayout as a root element.</p>
<p><img style="border: 1px solid black;" src="/images/stories/lessons/L0006/xL0006_015.jpg.pagespeed.ic.8e60S6aq7j.jpg" alt="" width="500" height="532"/></p>
<p> </p>
<p>Add three TableRow rows into the root TableLayout (from Layouts section on the left) and add two buttons into each row. The result: our table has got three rows and two columns.</p>
<p><img style="line-height: 1.3em; border: 1px solid black;" src="/images/stories/lessons/L0006/xL0006_020.jpg.pagespeed.ic._CyOnv21lK.jpg" alt="" width="561" height="302"/></p>
<p> </p>
<p>Add a few more buttons to the first row. The number of columns now equals 4, as it is defined by the row with the maximum number of elements and it is the first row in our case. For the second and the third rows the third and the fourth columns are empty.</p>
<p><img style="border: 1px solid black;" src="/images/stories/lessons/L0006/xL0006_030.jpg.pagespeed.ic.poJJ2tVC38.jpg" alt="" width="563" height="287"/></p>
<p> </p>
<p>Add a TextView and a Button to the second row and make text in the added TextView empty. Do the same with the third row. We can see that these elements are now placed in the third and the fourth column. And because TextView is empty and not seen on the screen, it seems like the third column in the second and third rows is empty.</p>
<p><img style="border: 1px solid black;" src="/images/stories/lessons/L0006/640x234xL0006_040.jpg.pagespeed.ic.cVCE6GsuoM.jpg" alt="" width="640" height="234"/></p>
<p> </p>
<p>The width of the column is defined by the widest element in this column. Enter some text into one of the TextViews and we can see that it has made a column wider.</p>
<p><img style="border: 1px solid black;" src="/images/stories/lessons/L0006/640x232xL0006_050.jpg.pagespeed.ic.q7P9SCPCFX.jpg" alt="" width="640" height="232"/></p>
<p> </p>
<p>Now I will remove elements of the fourth column and build a screen like this. Try to do it on your own as an exercise.</p>
<p><img style="border: 1px solid black;" src="/images/stories/lessons/L0006/xL0006_060.jpg.pagespeed.ic.TR_FfyT3Ui.jpg" alt="" width="335" height="319"/></p>
<p> </p>
<p>TL can contain not only TRs, but also simple Views. For example, add Button just into the TL, not into TR and you will see how it stretches to the width of the whole table.</p>
<p> </p>
<h3>RelativeLayout (RL)</h3>
<p> </p>
<p>In this type of Layout, each View-element can be positioned in a specific way relatively to the specified View-element.</p>
<p>Types of relationships:</p>
<p>1) to the left, right, above or below the specified element (layout_toLeftOf, layout_toRightOf, layout_above, layout_below)</p>
<p>2) aligned by the left, right, top or bottom edge of the specified element (layout_alignLeft, layout_alignRight, layout_alignTop, layout_alignBottom)</p>
<p>3) aligned by the left, right, top or bottom edge of a parent (layout_alignParentLeft, layout_alignParentRight, layout_alignParentTop, layout_alignParentBottom)</p>
<p>4) centered vertically, centered horizontally, centered vertically and horizontally relative to its parent (layout_centerVertical, layout_centerHorizontal, layout_centerInParent)</p>
<p>More details are available in <a href="http://developer.android.com/reference/android/widget/RelativeLayout.LayoutParams.html" target="_blank" rel="noopener noreferrer">help</a>.</p>
<p> </p>
<p>Create <b>rlayout.xml</b> and copy and paste this xml-code:</p>
<pre class="brush:xml">&lt;?xml version="1.0" encoding="utf-8"?&gt;
&lt;RelativeLayout
	xmlns:android="http://schemas.android.com/apk/res/android"
	android:layout_width="match_parent"
	android:layout_height="match_parent"&gt;
	&lt;TextView
		android:id="@+id/label"
		android:layout_width="match_parent"
		android:layout_height="wrap_content"
		android:text="Type here:"&gt;
	&lt;/TextView&gt;
	&lt;EditText
		android:id="@+id/entry"
		android:layout_width="match_parent"
		android:layout_height="wrap_content"
		android:layout_below="@+id/label"
		android:background="@android:drawable/editbox_background"&gt;
	&lt;/EditText&gt;
	&lt;Button
		android:id="@+id/ok"
		android:layout_width="wrap_content"
		android:layout_height="wrap_content"
		android:layout_alignParentRight="true"
		android:layout_below="@+id/entry"
		android:layout_marginLeft="10dip"
		android:text="OK"&gt;
	&lt;/Button&gt;
	&lt;Button
		android:layout_width="wrap_content"
		android:layout_height="wrap_content"
		android:layout_alignTop="@+id/ok"
		android:layout_toLeftOf="@+id/ok"
		android:text="Cancel"&gt;
	&lt;/Button&gt;
&lt;/RelativeLayout&gt;</pre>
<p>RelativeLayout is a the root element here.</p>
<p>We will get such a screen:</p>
<p><img style="border: 1px solid black;" src="/images/stories/lessons/L0006/xL0006_070.jpg.pagespeed.ic.7JizDqyY7p.jpg" alt="" width="367" height="534"/></p>
<p> </p>
<p>We are interested in xml-code. I will now shortly describe unknown attributes and their meanings:</p>
<p><span class="layoutFormatter_attrName">android:layout_width</span>=<span class="layoutFormatter_attrValue">"match_parent"</span> <br/><span class="layoutFormatter_attrName">android:layout_height</span>=<span class="layoutFormatter_attrValue">"wrap_content"</span> <br/><span class="layoutFormatter_attrName">android:id</span>=<span class="layoutFormatter_attrValue">"@+id/entry"</span></p>
<p>- the <b>android</b> word in the name of each element is a <a href="http://www.w3schools.com/xml/xml_namespaces.asp" target="_blank" rel="noopener noreferrer">namespace</a>, I will omit it in the explanations. <br/>- <b>id</b> - the element ID,<br/>- <b>layout_width</b> (the width of the element) and <b>layout_height</b> (the height of the element) can be specified in absolute values or can be the following: <b>fill_parent</b> (maximum available width or height in the bounds of its parent) and <b>wrap_content</b> (width of height is defined by the content of an element). Help states that there is also <b>match_parent</b>. It is the same as fill_parent. For some reasons, developers of the system decided that match_parent name is more convenient and they will stop using fill_parent soon. For now, it’s left for compatibility. So remember that <b>match_parent</b> = <b>fill_parent</b> and we will be trying to use <b>match_parent</b> further. Later we will stop on this and analyze it in more detail.</p>
<p>Now let’s get back to our elements. In the example we can see a TextView, EditText, and two Buttons - OK and Cancel. Let’s look through the attributes we are interested in.<br/><br/></p>
<p><b>TextView </b><br/><span class="layoutFormatter_attrName">android:id</span>=<span class="layoutFormatter_attrValue">"@+id/label"</span> - ID<br/><span class="layoutFormatter_attrName">android:layout_width</span>=<span class="layoutFormatter_attrValue">"match_parent"</span> - occupies all the available width (even though, it’s not visible on the screen) <br/><span class="layoutFormatter_attrName">android:layout_height</span>=<span class="layoutFormatter_attrValue">"wrap_content"</span> - height defined by the content <br/>it is not related to anything</p>
<p><b>EditText </b><br/><span class="layoutFormatter_attrName">android:id</span>=<span class="layoutFormatter_attrValue">"@+id/entry"</span> - ID<br/><span class="layoutFormatter_attrName">android:layout_width</span>=<span class="layoutFormatter_attrValue">"match_parent"</span> - occupies all the available width<br/><span class="layoutFormatter_attrName">android:layout_height</span>=<span class="layoutFormatter_attrValue">"wrap_content"</span> - height defined by the content<br/><span class="layoutFormatter_attrName">android:layout_below</span>=<span class="layoutFormatter_attrValue">"@id/label"</span> - located <b>below</b> the TextView (reference by ID)</p>
<p><b>Button_OK </b><br/><span class="layoutFormatter_attrName">android:id</span>=<span class="layoutFormatter_attrValue">"@+id/ok"</span> – ID<br/><span class="layoutFormatter_attrName">android:layout_width</span>=<span class="layoutFormatter_attrValue">"wrap_content"</span> - width defined by the content <br/><span class="layoutFormatter_attrName">android:layout_height</span>=<span class="layoutFormatter_attrValue">"wrap_content"</span> – height defined by the content <br/><span class="layoutFormatter_attrName">android:layout_below</span>=<span class="layoutFormatter_attrValue">"@id/entry"</span> - is located below EditText<br/><span class="layoutFormatter_attrName">android:layout_alignParentRight</span>=<span class="layoutFormatter_attrValue">"true"</span> - <strong>is aligned by the right edge of the parent</strong><br/><span class="layoutFormatter_attrName">android:layout_marginLeft</span>=<span class="layoutFormatter_attrValue">"10dip"</span> – has a margin on the left (for Button_Cancel not to be adjacent)</p>
<p><b>Button_Cancel</b><br/><span class="layoutFormatter_attrName">android:layout_width</span>=<span class="layoutFormatter_attrValue">"wrap_content"</span> - width defined by the content <br/><span class="layoutFormatter_attrName">android:layout_height</span>=<span class="layoutFormatter_attrValue">"wrap_content"</span> – height defined by the content <br/><span class="layoutFormatter_attrName">android:layout_toLeftOf</span>=<span class="layoutFormatter_attrValue">"@id/ok"</span> - located to the <b>left</b> of Button_OK<br/><span class="layoutFormatter_attrName">android:layout_alignTop</span>=<span class="layoutFormatter_attrValue">"@id/ok"</span> - <strong>aligned by the top edge of </strong>Button_OK</p>
<p> </p>
<p>You can add some more elements and experiment with their locations.</p>
<p>Pay your attention, that a View element can have no ID (android:id). For example, it is not usually needed for a TextView, because most of the times they are static and we rarely refer to them in the application. EditText is different - we work with the text field content, and a Button - we need to process clicks and to know exactly which Button has been pressed. In the future, we will see one more reason to specify an ID for a View-element</p>
<p> </p>
<h3>AbsoluteLayout (AL)</h3>
<p>Provides absolute positioning of elements on the screen. You specify the coordinates for the left top corner of the component.</p>
<p>Create <b>alayout.xml</b> with AbsoluteLayout as a root element</p>
<p><img style="border: 1px solid black;" src="/images/stories/lessons/L0006/xL0006_075.jpg.pagespeed.ic.w_BlzGdGXx.jpg" alt="" width="500" height="532"/></p>
<p> </p>
<p>Now try to add different elements by dragging them to the screen. They are not aligned as in LinearLayout or in TableLayout, but placed where you dragged them. So it is absolute positioning.</p>
<p><img style="border: 1px solid black;" src="/images/stories/lessons/L0006/xL0006_080.jpg.pagespeed.ic.Ow7auHBMUY.jpg" alt="" width="646" height="581"/></p>
<p> </p>
<p>Open xml-code and note that for specifying coordinates <b>layout_x</b> and <b>layout_y</b> properties are used</p>
<pre class="brush:xml">&lt;?xml version="1.0" encoding="utf-8"?&gt;
&lt;AbsoluteLayout
	xmlns:android="http://schemas.android.com/apk/res/android"
	android:layout_width="match_parent"
	android:layout_height="match_parent"&gt;
	&lt;Button
		android:id="@+id/button1"
		android:layout_width="wrap_content"
		android:layout_height="wrap_content"
		android:layout_x="42dp"
		android:layout_y="62dp"
		android:text="Button"&gt;
	&lt;/Button&gt;
	&lt;TextView
		android:id="@+id/textView1"
		android:layout_width="wrap_content"
		android:layout_height="wrap_content"
		android:layout_x="142dp"
		android:layout_y="131dp"
		android:text="TextView"&gt;
	&lt;/TextView&gt;
	&lt;CheckBox
		android:id="@+id/checkBox1"
		android:layout_width="wrap_content"
		android:layout_height="wrap_content"
		android:layout_x="55dp"
		android:layout_y="212dp"
		android:text="CheckBox"&gt;
	&lt;/CheckBox&gt;
	&lt;RadioButton
		android:id="@+id/radioButton1"
		android:layout_width="wrap_content"
		android:layout_height="wrap_content"
		android:layout_x="171dp"
		android:layout_y="318dp"
		android:text="RadioButton"&gt;
	&lt;/RadioButton&gt;
&lt;/AbsoluteLayout&gt;</pre>
<p> </p>
<p>First you might think that this is the most convenient and intuitive method of placing elements on the screen - they are placed wherever you want at once. But this is only in the case when you develop for the screen with a specific resolution. If you look at such application on the other screen, all the elements will be shifted and their positioning will be different from what you have planned. That’s why it is not recommended to use this Layout. And it’s compatibility with future Android versions is not guaranteed.</p>
<p>There are <a href="http://developer.android.com/guide/topics/ui/layout-objects.html#viewgroupsummary" target="_blank" rel="noopener noreferrer">lots</a> of ViewGroup types, and we will master them one by one. But for now, these are enough.</p>
<p><br/>In this lesson we have:<br/>- looked through main types of Layouts: LinearLayout, TableLayout, RelativeLayout, AbsoluteLayout</p>
<p>In the next lesson we will:<br/>- examine some Layout-properties of View-elements in detail, which allow us to configure their location in a ViewGroup.</p><div><hr>

	<section class="article-content clearfix">
		<p>В этом уроке:</p>
<p>- разбираем как можно использовать LayoutInflater</p>
 
<p> </p>
<p>После изучения SQLite самое время приступить к изучению списков – List. Но перед этим полезно будет узнать про <strong>LayoutInflater</strong>. Это знание пригодится нам в создании расширенных списков. Также перед этим уроком рекомендую снова прочесть урок про <a href="/ru/uroki/vse-uroki-spiskom/49-16-layoutparams.html">LayoutParams</a>, освежить знания.</p>
<p><a href="http://developer.android.com/reference/android/view/LayoutInflater.html" target="_blank">LayoutInflater</a> – это класс, который умеет из содержимого layout-файла создать View-элемент. Метод который это делает называется <strong>inflate</strong>. Есть несколько реализаций этого метода с различными параметрами. Но все они используют друг друга и результат их выполнения один – <strong>View</strong>.</p>
<p>Мы рассмотрим эту реализацию – <a href="http://developer.android.com/reference/android/view/LayoutInflater.html#inflate(int, android.view.ViewGroup, boolean)" target="_blank">public View inflate (int resource, ViewGroup root, boolean attachToRoot)</a> </p>
<p>Как видим, на вход метод принимает три параметра:</p>
<p><strong>resource</strong> - ID layout-файла, который будет использован для создания View. Например - R.layout.main<br/><strong>root</strong> – родительский ViewGroup-элемент для создаваемого View. LayoutParams от этого ViewGroup присваиваются создаваемому View.<br/><strong>attachToRoot</strong> – присоединять ли создаваемый View к root. Если true, то root становится родителем создаваемого View. Т.е. это равносильно команде root.addView(View).  Если false – то создаваемый View просто получает LayoutParams от root, но его дочерним элементом не становится.</p>
<p> </p>
<p>Посмотрим на практике.</p>
<p> </p>
<p>Создадим проект:</p>
<p><strong>Project name</strong>: P0401_LayoutInflater<br/> <strong>Build Target</strong>: Android 2.3.3<br/> <strong>Application name</strong>: LayoutInflater<br/> <strong>Package name</strong>: ru.startandroid.develop.p0401layoutinflater<br/> <strong>Create Activity</strong>: MainActivity</p>
<p> </p>
<p>Открываем <strong>main.xml</strong> и рисуем такой экран:</p>
<pre class="brush:xml">&lt;?xml version="1.0" encoding="utf-8"?&gt;
&lt;LinearLayout
    xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="fill_parent"
    android:layout_height="fill_parent"
    android:orientation="vertical"&gt;
    &lt;LinearLayout
        android:id="@+id/linLayout"
        android:layout_width="match_parent"
        android:layout_height="wrap_content"
        android:layout_margin="20dp"&gt;
        &lt;TextView
            android:layout_width="wrap_content"
            android:layout_height="wrap_content"
            android:text="Linear Layout: "&gt;
        &lt;/TextView&gt;
    &lt;/LinearLayout&gt;
    &lt;RelativeLayout
        android:id="@+id/relLayout"
        android:layout_width="match_parent"
        android:layout_height="wrap_content"
        android:layout_margin="20dp"&gt;
        &lt;TextView
            android:layout_width="wrap_content"
            android:layout_height="wrap_content"
            android:text="Relative Layout: "&gt;
        &lt;/TextView&gt;
    &lt;/RelativeLayout&gt;
&lt;/LinearLayout&gt;</pre>
<p>На экране две ViewGroup - <strong>linLayout </strong>и <strong>relLayout</strong>. В них по TextView с соответствующим текстом.</p>
<p> </p>
<p>Создадим еще один layout-файл<strong> text.xml</strong>:</p>
<pre class="brush:xml">&lt;?xml version="1.0" encoding="utf-8"?&gt;
&lt;TextView
    xmlns:android="http://schemas.android.com/apk/res/android"
    android:id="@+id/tvLayout"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:text="Layout with TextView"&gt;
&lt;/TextView&gt;</pre>
<p>Тут просто <strong>TextView </strong>без всяких <strong>ViewGroup</strong>. На нем мы и будем испытывать <strong>LayoutInflater</strong>.</p>
<p> </p>
<p> </p>
<p>Открываем <strong>MainActivity.java</strong> и пишем код:</p>
<div class="java">
<pre class="brush:java">package ru.startandroid.develop.p0401layoutinflater;

import android.app.Activity;
import android.os.Bundle;
import android.util.Log;
import android.view.LayoutInflater;
import android.view.View;
import android.view.ViewGroup.LayoutParams;
import android.widget.TextView;

public class MainActivity extends Activity {
  
  final String LOG_TAG = "myLogs";
  
    /** Called when the activity is first created. */
    public void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.main);
        
        LayoutInflater ltInflater = getLayoutInflater();
        View view = ltInflater.inflate(R.layout.text, null, false);
        LayoutParams lp = view.getLayoutParams();
        
        Log.d(LOG_TAG, "Class of view: " + view.getClass().toString());
        Log.d(LOG_TAG, "LayoutParams of view is null: " + (lp == null));
        Log.d(LOG_TAG, "Text of view: " + ((TextView) view).getText());
    }
}</pre>
<p>Мы получаем <strong>LayoutInflater</strong> методом <a href="http://developer.android.com/reference/android/app/Activity.html#getLayoutInflater()" target="_blank">getLayoutInflater</a>, используем его для получения <strong>View</strong>-элемента из layout-файла<strong> text.xml</strong> и считываем <strong>LayoutParams </strong>у свежесозданного view.</p>
</div>
<p>Обратите внимание на параметры, которые мы использовали для метода <strong>inflate</strong>. Мы указали <strong>ID </strong>layout-ресурса, передали <strong>null </strong>в качестве родительского элемента и, соответственно, привязка к родителю - <strong>false</strong>.</p>
<p>Все сохраним и запустим.</p>
<p> </p>
<p>На экране ничего не изменилось. Т.к. мы конвертнули layout в view, но никуда его не поместили. Он просто висит в памяти.</p>
<p><img src="https://lh4.googleusercontent.com/-2Qy9EpQksok/TtPlEA7wM4I/AAAAAAAAApk/dgTNGaeUJ2E/s640/20111127_L0040_L_First.jpg" alt="" width="640" height="454" border="0"/></p>
<p> </p>
<p>Смотрим лог:</p>
<p><em>Class of view: class android.widget.TextView<br/></em><em>LayoutParams of view is null: true<br/></em><em>Text of view: Layout with TextView</em></p>
<p>Мы видим класс созданного элемента - <strong>TextView</strong>. Все верно - этот элемент и был в файле <strong>text.xml</strong>. Далее видим <strong>null </strong>вместо <strong>LayoutParams</strong>. Это произошло потому, что родителя в методе inflate мы указали <strong>null</strong>. А именно от родителя <strong>view </strong>и должен был получить <strong>LayoutParams</strong>. Третья строка лога показывает текст TextView. Он тот же, что и в layout-файле <strong>text.xml</strong> – все верно.</p>
<p> </p>
<p>Давайте немного изменим программу. Будем добавлять наш созданный элемент в <strong>linLayout</strong> из <strong>main.xml</strong>. Делается это просто – командой <strong>addView</strong>.</p>
<div class="java">
<pre class="brush:xml; highlight: [3,4]">LayoutParams lp = view.getLayoutParams();
        
LinearLayout linLayout = (LinearLayout) findViewById(R.id.linLayout);
linLayout.addView(view);
        
Log.d(LOG_TAG, "Class of view: " + view.getClass().toString());  </pre>
</div>
<p>(добавляете только выделенные строки)</p>
<p> </p>
<p>Мы нашли <strong>linLayout </strong>с экрана и добавили в него созданный с помощью LayoutInflater элемент.</p>
<p> </p>
<p>Сохраняем, запускаем. Видим, что элемент добавился на экран в linLayout.</p>
<p><img src="https://lh3.googleusercontent.com/-t5hhgilAV1g/TtPlDIBn1LI/AAAAAAAAApc/yV16mGka4Dw/s640/20111127_L0040_L_Second.jpg" alt="" width="640" height="454" border="0"/></p>
<p> </p>
<p>Теперь давайте попробуем указать родителя (<strong>root</strong>) при вызове метода <strong>inflate</strong>. Перепишем метод <strong>onCreate</strong>:</p>
<div class="java">
<pre class="brush:java">  public void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.main);

    LayoutInflater ltInflater = getLayoutInflater();

    LinearLayout linLayout = (LinearLayout) findViewById(R.id.linLayout);
    View view1 = ltInflater.inflate(R.layout.text, linLayout, false);
    LayoutParams lp1 = view1.getLayoutParams();

    Log.d(LOG_TAG, "Class of view1: " + view1.getClass().toString());
    Log.d(LOG_TAG, "Class of layoutParams of view1: " + lp1.getClass().toString());
    Log.d(LOG_TAG, "Text of view1: " + ((TextView) view1).getText());

    RelativeLayout relLayout = (RelativeLayout) findViewById(R.id.relLayout);
    View view2 = ltInflater.inflate(R.layout.text, relLayout, false);
    LayoutParams lp2 = view2.getLayoutParams();

    Log.d(LOG_TAG, "Class of view2: " + view2.getClass().toString());
    Log.d(LOG_TAG, "Class of layoutParams of view2: " + lp2.getClass().toString());
    Log.d(LOG_TAG, "Text of view2: " + ((TextView) view2).getText());
  }</pre>
<p> </p>
</div>
<p>Мы находим элементы <strong>linLayout </strong>и <strong>relLayout </strong>с экрана и с помощью <strong>LayoutInflater </strong>создаем два View-элемента из layout-файла text.xml. Для <strong>первого </strong>указываем root – <strong>linLayout</strong>, для <strong>второго </strong>– <strong>relLayout</strong>. Но третий параметр <strong>attachToRoot</strong> оставляем <strong>false</strong>. Это значит, что созданный <strong>View</strong>-элемент <strong>получит LayoutParams </strong>от <strong>root</strong>-элемента, но <strong>не добавится</strong> к нему.</p>
<p> </p>
<p>Все сохраним, запустим. На экране ничего не поменялось. Т.к. мы ни к чему новые элементы не добавляли и <strong>attachToRoot </strong>= <strong>false</strong>.</p>
<p><img src="https://lh4.googleusercontent.com/-2Qy9EpQksok/TtPlEA7wM4I/AAAAAAAAApk/dgTNGaeUJ2E/s640/20111127_L0040_L_First.jpg" alt="" width="640" height="454" border="0"/></p>
<p> </p>
<p>Смотрим лог:</p>
<p><em>Class of view1: class android.widget.TextView<br/></em><em>Class of layoutParams of view1: class android.widget.LinearLayout$LayoutParams<br/></em><em>Text of view1: Layout with TextView<br/></em><em>Class of view2: class android.widget.TextView<br/></em><em>Class of layoutParams of view2: class android.widget.RelativeLayout$LayoutParams<br/></em><em>Text of view2: Layout with TextView</em></p>
<p>По логам видно, что класс созданных элементов – <strong>TextView</strong>. А класс <strong>LayoutParams </strong>различается. В <strong>первом </strong>случае – это <strong>LinearLayout$LayoutParams</strong>, т.к. в качестве <strong>root </strong>элемента в методе inflate мы указали <strong>linLayout</strong>, а это объект класса <strong>LinearLayout</strong>. Во втором случае класс <strong>LayoutParams </strong>у созданного элемента - <strong>RelativeLayout$LayoutParams</strong>. Потому, что в качестве root указали <strong>relLayout </strong>(класс <strong>RelativeLayout</strong>).</p>
<p> </p>
<p>Теперь у нас два варианта, как добавить созданные <strong>view1 </strong>и <strong>view2 </strong>на экран.</p>
<p>1) Снова использовать методы <strong>addView</strong></p>
<p>2) Передавать <strong>true </strong>в качестве третьего параметра метода <strong>inflate</strong>. Тогда созданный <strong>View</strong>-элемент будет добавлен к root.</p>
<p> </p>
<p>Выберем второй вариант и внесем изменения в код:</p>
<div class="java">
<pre class="brush:java">  public void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.main);

    LayoutInflater ltInflater = getLayoutInflater();

    LinearLayout linLayout = (LinearLayout) findViewById(R.id.linLayout);
    View view1 = ltInflater.inflate(R.layout.text, linLayout, true);
    LayoutParams lp1 = view1.getLayoutParams();

    Log.d(LOG_TAG, "Class of view1: " + view1.getClass().toString());
    Log.d(LOG_TAG, "Class of layoutParams of view1: " + lp1.getClass().toString());

    RelativeLayout relLayout = (RelativeLayout) findViewById(R.id.relLayout);
    View view2 = ltInflater.inflate(R.layout.text, relLayout, true);
    LayoutParams lp2 = view2.getLayoutParams();

    Log.d(LOG_TAG, "Class of view2: " + view2.getClass().toString());
    Log.d(LOG_TAG, "Class of layoutParams of view2: " + lp2.getClass().toString());
  }</pre>
<p>Передаем <strong>true </strong>в качестве <strong>третьего </strong>параметра в методе <strong>inflate </strong>и убираем строки выведения в лог текстов из TextView. Сейчас будет понятно почему.</p>
</div>
<p>Все сохраним и запустим приложение.</p>
<p><img src="https://lh3.googleusercontent.com/-kBpMC91eChI/TtPlEEujttI/AAAAAAAAApo/4n5o-JEFAes/s640/20111127_L0040_L_Third.jpg" alt="" width="640" height="454" border="0"/></p>
<p> </p>
<p>Как видим, созданные <strong>TextView </strong>появились в своих родителях, которых мы указали в методе <strong>inflate</strong>. В <strong>RelativeLayout </strong>элементы наложились друг на друга, т.к. мы не настроили расположение. В данный момент это не существенно. </p>
<p> </p>
<p>Смотрим лог:</p>
<p><em>Class of view1: class android.widget.LinearLayout<br/> Class of layoutParams of view1: class android.widget.LinearLayout$LayoutParams<br/> Class of view2: class android.widget.RelativeLayout<br/> Class of layoutParams of view2: class android.widget.LinearLayout$LayoutParams</em></p>
<p>Обратите внимание на класс элементов. В первом случае - это <strong>LinearLayout</strong>, а во втором - <strong>RelativeLayout</strong>. Т.е. метод <strong>inflate </strong>вернул нам <strong>не созданные</strong> из layout-файла View-элементы, а те, что мы указывали как <strong>root</strong>. А <strong>созданные </strong>из layout-файла <strong>View </strong>элементы он <strong>добавил </strong>в <strong>root </strong>как дочерние аналогично команде <strong>addView</strong>. Это произошло потому, что мы указали <strong>true </strong>в третьем параметре (attachToRoot) метода inflate. </p>
<p>Соответственно <strong>LayoutParams </strong>для <strong>view1 </strong>и <strong>view2 </strong>будет<strong> LinearLayout$LayoutParams</strong>, т.к. <strong>linLayout </strong>и <strong>relLayout </strong>имеют родителя <strong>LinearLayout</strong>. И <strong>LayoutParams </strong>берут от него.</p>
<p>Для закрепления темы на следующем уроке сделаем пример поинтереснее.</p>
<p> </p>
